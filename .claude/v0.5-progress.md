### CP-3: patchBodyChildren off-by-one fix ✅
- Replaced the `recognizedIdx--` backtracking pattern with a look-ahead approach
- On type mismatch, checks if the *next* DOM child matches the current Lexical node
- If look-ahead matches: removes current DOM child, lets loop advance to the matching one
- If no look-ahead match: replaces current DOM child with new element and advances lexIdx
- This prevents the bug where consecutive mismatches caused Lexical nodes to be compared against the wrong DOM children

### CP-5: Fix savedXmlRef mutation inside setTabs updaters ✅
- Moved `savedXmlRef.current = content` mutations out of `setTabs(prev => ...)` functional updaters
- 5 mutation sites fixed across 2 files:
  - `hooks/useHerettoCms.ts`: `handleHerettoSave`, `doHerettoRefresh`, `handleHerettoSaveNew`
  - `hooks/useLocalFile.ts`: `handleSaveConfirm` (showSaveFilePicker path + fallback download path)
- Approach: `savedXmlRef` is a stable ref-like object created once per tab in `createTab()`. Mutating `.current` outside the updater is safe — the same object is referenced regardless of React re-renders.
- For `doHerettoRefresh`: added `savedXmlRef` as a third parameter (callers already have the tab object)
