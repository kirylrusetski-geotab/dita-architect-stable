### CP-3: patchBodyChildren off-by-one fix ✅
- Replaced the `recognizedIdx--` backtracking pattern with a look-ahead approach
- On type mismatch, checks if the *next* DOM child matches the current Lexical node
- If look-ahead matches: removes current DOM child, lets loop advance to the matching one
- If no look-ahead match: replaces current DOM child with new element and advances lexIdx
- This prevents the bug where consecutive mismatches caused Lexical nodes to be compared against the wrong DOM children
